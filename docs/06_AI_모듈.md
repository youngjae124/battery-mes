## 1. Rule-based SPC 분석

`POST /analysis/spc` — 샘플값 리스트를 받아 통계 지표를 계산합니다.

### 계산 항목
- 샘플 수, 평균, 최대·최소, 표준편차
- 공정 상태 판정: 표준편차 ≤ 0.2 → `NORMAL`, 초과 → `WARNING`
- USL/LSL 입력 시 Cp/Cpk 공정능력지수 계산

### Cp/Cpk 산식
- `Cp = (USL - LSL) / (6 * σ)`
- `Cpk = min((USL - μ) / (3 * σ), (μ - LSL) / (3 * σ))`
- σ는 모집단 표준편차(`ddof=0`) 사용 (R-bar/d2 기반 정식 Cpk의 근사치)
- `usl`/`lsl` 누락, `usl <= lsl`, `σ = 0` 시 `cp`/`cpk` → `null`

## 4. VLM — 불량 이미지 분석 (개발 예정)

### 개요
불량 등록 시 사진을 첨부하면 Gemini Vision이 이미지를 보고 불량 유형·위치·원인을 시각적으로 분석합니다.
Gemini는 기본적으로 멀티모달이므로 **별도 모델 추가 없이** 기존 `gemini-2.5-flash-lite`에 이미지를 함께 전송하면 됩니다.

### 계획 아키텍처
```
React (이미지 업로드)
  → Spring Boot POST /api/analysis/defect-image (multipart)
  → FastAPI POST /analysis/defect-image
  → Gemini API (이미지 + 텍스트 프롬프트)
  → 한국어 시각 분석 텍스트 반환
```

### 구현 계획
1. **프론트엔드**: 불량 등록 폼에 이미지 업로드 필드 추가, base64 변환 후 전송
2. **FastAPI**: `/analysis/defect-image` 엔드포인트 — 이미지(base64) + 불량 메타데이터를 Gemini에 전달
3. **Spring Boot**: 멀티파트 요청을 Python 서비스로 프록시
