# 04. 시스템 아키텍처

## 개요

기존 MES는 Java(Spring Boot) + React + PostgreSQL 중심 구조입니다.
이번 작업으로 Python(FastAPI) 분석 서비스를 별도 마이크로서비스로 분리했습니다.

## 서비스 역할

### Frontend
운영자 화면, 생산/설비/품질/SPC/보고서 입력 및 조회. Recharts 기반 차트 시각화(대시보드·SPC 관리도·보고서·불량 분석)와 exceljs/jsPDF 기반 Excel·PDF 내보내기 기능을 포함합니다.

### Backend
권한, 비즈니스 로직, PostgreSQL 접근, Python 분석 서비스 브리지 역할

### Python Analysis
SPC 분석(평균·표준편차·Cp/Cpk 계산), 향후 AI 모델 서빙 전용 서비스

### PostgreSQL
MES 원천 데이터 저장소

## 아키텍처 흐름

1. 사용자가 React 화면에서 요청합니다.
2. Spring Boot가 인증/권한과 업무 로직을 처리합니다.
3. 분석이 필요한 경우 Spring Boot가 Python 분석 서비스에 HTTP 요청을 보냅니다.
4. Python 서비스가 계산 결과를 응답합니다.
5. Spring Boot가 이를 표준 JSON 응답으로 프론트에 전달합니다.
